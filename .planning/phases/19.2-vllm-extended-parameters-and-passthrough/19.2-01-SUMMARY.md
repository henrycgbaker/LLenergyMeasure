## Accomplishments

- Added `VLLMAttentionConfig` with 10 typed fields covering all vLLM attention backend options (flash_attn, FlashInfer, cuDNN, TRT-LLM); uses `extra="allow"` for forward compat
- Added `VLLMBeamSearchConfig` with 4 typed fields (beam_width, length_penalty, early_stopping, max_tokens); wired as `VLLMConfig.beam_search` with mutual-exclusion guard against `sampling`
- Changed `extra="forbid"` to `extra="allow"` on all 5 backend config models, enabling researchers to pass any upstream API field without schema updates
- Added 7 new `VLLMEngineConfig` fields covering CPU offload, multi-GPU, absolute KV cache sizing, and compilation config passthrough
- Added `n` field to `VLLMSamplingConfig` for multi-sequence generation

## Decisions Made

- `extra="allow"` on all backend configs — unknown YAML fields stored in `model_extra`, forwarded to native API at runtime in Plan 02 wiring
- `VLLMAttentionConfig.backend` is `str` not `Literal` — vLLM owns the validation of attention backend names, not our schema
- `compilation_config` is `dict[str, Any]` full passthrough — vLLM's CompilationConfig has ~30 internal fields, coupling to them would create fragile upstream dependency
- `offload_params` typed as `list[str]` in YAML — YAML-friendly; Plan 02 will convert to `set[str]` when forwarding to vLLM if needed
