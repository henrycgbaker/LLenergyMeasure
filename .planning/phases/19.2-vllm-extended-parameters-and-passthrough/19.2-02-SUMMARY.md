## Decisions Made

- Beam search branching implemented at the `_run_measurement()` call site via `isinstance(sampling_params, BeamSearchParams)` — sampling_params type carries routing information, no need for separate flag
- `engine.model_extra` merged LAST in `_build_llm_kwargs()` — deliberate design: user passthrough extras can override explicit fields (escape hatch for vLLM API changes)
- `attention.backend` -> `attention_backend` name mapping applied at wiring time, not schema level — schema uses vLLM-agnostic name, wiring handles the API-specific rename
- `offload_params` converts `list[str]` -> `set[str]` at wiring time — YAML list is user-friendly, vLLM API expects set[str]
